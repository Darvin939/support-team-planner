# Support Team Planner

Веб-приложение для планирования и отслеживания задач команд поддержки. Позволяет распределять задачи между сотрудниками,
управлять статусами выполнения, фиксировать дни фриза и просматривать статистику.

---

## Возможности

- **Команды** — создание, редактирование и удаление команд с привязкой разрешённых шаблонов блоков раскатки.
- **Блоки раскатки** — именованные этапы деплоя (например, ГФ, Б1). Названия хранятся в верхнем регистре.
- **Шаблоны блоков** — наборы блоков со сдвигом в днях от базовой даты назначения; команде разрешается один или
  несколько шаблонов, что определяет, по какому графику для неё работает автоназначение.
- **Сотрудники** — добавление и редактирование сотрудников с ФИО.
- **Задачи** — создание задач с уровнем критичности (высокая / средняя / низкая) и зависимостями между задачами
  (с проверкой на циклы).
- **Планирование** — назначение задач на даты, распределение по сотрудникам, указание блоков, комментариев и
  маркера ПСИ. Автоназначение по графику блоков команды. Таблица автоматически прокручивается к текущему дню.
- **Статусы назначений** — новый → запланирован → успешно / откат.
- **Статусы задач** — новый → к планированию → в работе → выполнено / отменено. Задачи в терминальных статусах (
  выполнено/отменено) недоступны для редактирования.
- **Поиск задач** — нечёткий поиск по названию/описанию с допуском на опечатки (посимвольное скользящее окно).
- **Дни фриза** — добавление отдельных дат, диапазонов или замена целиком за месяц через модальное окно.
- **Статистика** — таблицы активных работ (new/planned) за период и на сегодня со счётчиками по статусам и критичности.
- **Сохранение фильтров** — выбранная команда и период дат сохраняются в localStorage и синхронизируются между
  страницами планирования и статистики.

## Быстрый старт

```bash
# 1. Установите зависимости
pip install -r requirements.txt

# 2. Запустите приложение
python support_planner.py
```

При первом запуске автоматически создастся файл `database.db` с необходимыми таблицами.

Откройте браузер: [http://localhost:5093](http://localhost:5093)

Для продакшен-запуска (с проверкой, что процесс уже не запущен, и логированием) используются `check.sh` и `run.sh`:

```bash
./check.sh   # запускает run.sh в фоне через nohup, если процесс ещё не работает
```

## API

Все эндпоинты доступны через `/api/` и возвращают JSON.

### Команды

| Метод  | Путь              | Описание                         |
|--------|-------------------|----------------------------------|
| GET    | `/api/teams`      | Список команд с их шаблонами     |
| GET    | `/api/teams/{id}` | Команда с разрешёнными шаблонами |
| POST   | `/api/teams`      | Создать команду                  |
| PUT    | `/api/teams/{id}` | Обновить команду                 |
| DELETE | `/api/teams/{id}` | Удалить команду                  |

### Сотрудники

| Метод  | Путь                  | Описание            |
|--------|-----------------------|---------------------|
| GET    | `/api/employees`      | Список сотрудников  |
| POST   | `/api/employees`      | Создать сотрудника  |
| PUT    | `/api/employees/{id}` | Обновить сотрудника |
| DELETE | `/api/employees/{id}` | Удалить сотрудника  |

### Задачи

| Метод  | Путь                               | Описание                                                                                   |
|--------|------------------------------------|--------------------------------------------------------------------------------------------|
| GET    | `/api/tasks/{team_id}`             | Задачи команды (`?search=&offset=&limit=&show_completed=`, дефолт 20/стр., нечёткий поиск) |
| POST   | `/api/task`                        | Создать / обновить задачу (тело может включать `dependency_ids`)                           |
| DELETE | `/api/task/{id}`                   | Удалить задачу                                                                             |
| PATCH  | `/api/tasks/{id}/status`           | Обновить статус задачи (тело: `{"status": "..."}`)                                         |
| GET    | `/api/tasks/{team_id}/deps`        | Зависимости всех задач команды                                                             |
| GET    | `/api/tasks/{team_id}/active-list` | Плоский список активных задач команды (для выбора зависимостей)                            |

### Назначения

| Метод  | Путь                         | Описание                                               |
|--------|------------------------------|--------------------------------------------------------|
| GET    | `/api/assignments/{team_id}` | Назначения команды (фильтр: `?start_date=&end_date=`)  |
| POST   | `/api/assignment`            | Создать / обновить назначение (тело включает `is_psi`) |
| DELETE | `/api/assignment/{id}`       | Удалить назначение                                     |

### Блоки и шаблоны блоков

| Метод  | Путь                        | Описание                              |
|--------|-----------------------------|---------------------------------------|

| POST   | `/api/blocks`               | Создать блок                          |
| DELETE | `/api/blocks/{id}`          | Удалить блок                          |
| GET    | `/api/block-templates`      | Список всех шаблонов блоков с блоками |
| POST   | `/api/block-templates`      | Создать шаблон блоков                 |
| PUT    | `/api/block-templates/{id}` | Обновить шаблон блоков                |
| DELETE | `/api/block-templates/{id}` | Удалить шаблон блоков                 |

### Дни фриза

| Метод  | Путь                                    | Описание                            |
|--------|-----------------------------------------|-------------------------------------|
| GET    | `/api/freeze-days`                      | Список дней фриза                   |
| POST   | `/api/freeze-days`                      | Добавить день или диапазон          |
| DELETE | `/api/freeze-days/{date}`               | Удалить день                        |
| PUT    | `/api/freeze-days/month`                | Заменить дни фриза за месяц целиком |

### Активные назначения (статистика)

| Метод | Путь                                | Описание                                                                                         |

## База данных

**Движок:** SQLite по умолчанию (файл `database.db`, автоматическое создание при первом запуске). Слой `db/`
абстрагирован от конкретной СУБД через `DBBackend`; в `db/postgres.py` реализован альтернативный `PostgresBackend`
(требует `psycopg2-binary`), который можно подключить, заменив `_backend` в `db/__init__.py`.

**Таблицы:**

- `teams` — команды
- `blocks` — блоки раскатки
- `block_templates` + `template_blocks` — шаблоны блоков и их состав (блок + сдвиг в днях)
- `team_templates` — связь команда ↔ разрешённые шаблоны
- `employees` — сотрудники
- `freeze_days` — дни фриза
- `tasks` — задачи (привязаны к команде, каскадное удаление; колонка `task_status`)
- `task_dependencies` — зависимости между задачами (граф, проверяется на циклы при добавлении)
- `assignments` — назначения (связь задачи, даты и сотрудника; уникальность по task_id + date; флаг `is_psi`)

**Примечание:** `database.db` добавлен в `.gitignore` и не хранится в репозитории.

## Разработка

- Приложение запускается через Uvicorn на порту 5093.
- При удалении сотрудника с активными назначениями они открепляются (employee_id → NULL), но не удаляются.
- Период в фильтрах ограничен 60 днями, даты — диапазоном 2000–2099.
- Поиск задач нечувствителен к опечаткам (до ~1 ошибки на 7 символов слова).
- Интерфейс выполнен на русском языке.
