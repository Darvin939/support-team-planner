# 📋 Support Team Planner

Веб-приложение для планирования и отслеживания задач команд поддержки. Позволяет распределять задачи между сотрудниками,
управлять статусами выполнения, фиксировать дни фриза и просматривать статистику.

---

## Возможности

- **Команды** — создание, редактирование и удаление команд.
- **Сотрудники** — добавление и редактирование сотрудников с ФИО.
- **Задачи** — создание задач с уровнем критичности (высокая / средняя / низкая).
- **Планирование** — назначение задач на даты, распределение по сотрудникам, указание блоков (часов/смен) и
  комментариев.
- **Статусы** — новый → запланирован → успешно / откат.
- **Дни фриза** — добавление отдельных дат или диапазонов (периоды заморозки изменений).
- **Статистика** — общая статистика по активным задачам, распределение по критичности, статусы на сегодня, ежедневная
  динамика.

## Быстрый старт

```bash
# 1. Установите зависимости
pip install -r requirements.txt

# 2. Запустите приложение
python app.py
```

При первом запуске автоматически создастся файл `database.db` с необходимыми таблицами.

Откройте браузер: [http://localhost:5000](http://localhost:5000)

## API

Все эндпоинты доступны через `/api/` и возвращают JSON.

### Команды

| Метод  | Путь              | Описание         |
|--------|-------------------|------------------|
| GET    | `/api/teams`      | Список команд    |
| POST   | `/api/teams`      | Создать команду  |
| PUT    | `/api/teams/<id>` | Обновить команду |
| DELETE | `/api/teams/<id>` | Удалить команду  |

### Сотрудники

| Метод  | Путь                  | Описание            |
|--------|-----------------------|---------------------|
| GET    | `/api/employees`      | Список сотрудников  |
| POST   | `/api/employees`      | Создать сотрудника  |
| PUT    | `/api/employees/<id>` | Обновить сотрудника |
| DELETE | `/api/employees/<id>` | Удалить сотрудника  |

### Задачи

| Метод  | Путь                   | Описание                  |
|--------|------------------------|---------------------------|
| GET    | `/api/tasks/<team_id>` | Задачи команды            |
| POST   | `/api/task`            | Создать / обновить задачу |
| DELETE | `/api/task/<id>`       | Удалить задачу            |

### Назначения

| Метод  | Путь                         | Описание                                              |
|--------|------------------------------|-------------------------------------------------------|
| GET    | `/api/assignments/<team_id>` | Назначения команды (фильтр: `?start_date=&end_date=`) |
| POST   | `/api/assignment`            | Создать / обновить назначение                         |
| DELETE | `/api/assignment/<id>`       | Удалить назначение                                    |

### Дни фриза

| Метод  | Путь                      | Описание                   |
|--------|---------------------------|----------------------------|
| GET    | `/api/freeze-days`        | Список дней фриза          |
| POST   | `/api/freeze-days`        | Добавить день или диапазон |
| DELETE | `/api/freeze-days/<date>` | Удалить день               |

### Статистика

| Метод | Путь                         | Описание                                       |
|-------|------------------------------|------------------------------------------------|
| GET   | `/api/statistics/<team_id>`  | Статистика команды (0 = все команды)           |
| GET   | `/api/daily-stats/<team_id>` | Ежедневная динамика (`?start_date=&end_date=`) |

## База данных

**Движок:** SQLite (файл `database.db`, автоматическое создание при первом запуске).

**Таблицы:**

- `teams` — команды
- `employees` — сотрудники
- `freeze_days` — дни фриза
- `tasks` — задачи (привязаны к команде)
- `assignments` — назначения (связь задачи, даты и сотрудника)

**Примечание:** `database.db` добавлен в `.gitignore` и не хранится в репозитории.

## Разработка

- Приложение работает в debug-режиме на порту 5000.
- При удалении сотрудника с активными назначениями они открепляются, но не удаляются.
- Код не использует сторонних линтеров / форматтеров.
- Интерфейс выполнен на русском языке.
