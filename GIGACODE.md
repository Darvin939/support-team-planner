# Support Team Planner

## Описание проекта

**Support Team Planner** — это веб-приложение на Flask для планирования и отслеживания задач команд поддержки. Позволяет
распределять задачи между сотрудниками, управлять статусами выполнения, фиксировать «дни фриза» (периоды, когда
изменения не производятся) и просматривать статистику.

### Основные возможности

- **Управление командами:** создание, редактирование и удаление команд.
- **Управление сотрудниками:** добавление и редактирование сотрудников с ФИО.
- **Задачи:** создание задач с указанием критичности (high/medium/low) и привязкой к команде.
- **Планирование:** drag-and-drop (судя по объёмному `script.js`) назначение задач на даты, распределение по
  сотрудникам, указание блоков (часов/смен), комментариев.
- **Статусы назначений:** `new` — новый, `planned` — запланирован, `rollback` — откат, `success` — успешно.
- **Дни фриза:** добавление отдельных дат или диапазонов, когда работы не производятся.
- **Статистика:** общая статистика активных задач, распределение по критичности, статусы на сегодня, ежедневная
  динамика.

## Стек технологий

| Компонент         | Технология                  |
|-------------------|-----------------------------|
| Backend           | Python 3, Flask 2.3.3       |
| База данных       | SQLite (файл `database.db`) |
| Шаблоны           | Jinja2 (HTML)               |
| Стили             | CSS (один файл `style.css`) |
| Клиентская логика | Vanilla JS (`script.js`)    |

## Архитектура

```
app.py           — маршруты (роуты) и API-эндпоинты
database.py      — работа с БД (SQLite), DAO-слой с декоратором @with_db_connection
utils.py         — вспомогательные функции (форматирование ФИО)
templates/       — Jinja2-шаблоны
static/css/      — стили
static/js/       — клиентский JavaScript
```

Кодовая база следует паттерну **«толстый монолит»** (один Flask app, DAO и контроллеры в двух файлах). API возвращает
JSON для асинхронных запросов со страниц планирования и статистики.

## Запуск

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск (режим debug, порт 5000)
python app.py
```

При первом запуске автоматически создаётся файл `database.db` с таблицами (через `init_db()` в `database.py`).

## Структура БД (SQLite)

- **teams** — команды (id, name)
- **employees** — сотрудники (id, last_name, first_name, middle_name)
- **freeze_days** — дни фриза (id, date)
- **tasks** — задачи (id, team_id, name, description, criticality)
- **assignments** — назначения (id, task_id, date, block, status, employee_id, comment)

Связи: tasks → teams, assignments → tasks, assignments → employees. Каскадное удаление при удалении команды или задачи.

## API-эндпоинты (все через `/api/`)

### Команды

- `GET /api/teams` — список команд
- `POST /api/teams` — создать команду
- `PUT /api/teams/<id>` — обновить команду
- `DELETE /api/teams/<id>` — удалить команду

### Сотрудники

- `GET /api/employees` — список сотрудников
- `POST /api/employees` — создать сотрудника
- `PUT /api/employees/<id>` — обновить сотрудника
- `DELETE /api/employees/<id>` — удалить сотрудника

### Задачи

- `GET /api/tasks/<team_id>` — задачи команды
- `POST /api/task` — создать/обновить задачу
- `DELETE /api/task/<id>` — удалить задачу

### Назначения

- `GET /api/assignments/<team_id>` — назначения команды (с фильтром по датам `?start_date=&end_date=`)
- `POST /api/assignment` — создать/обновить назначение
- `DELETE /api/assignment/<id>` — удалить назначение

### Дни фриза

- `GET /api/freeze-days` — список дней фриза
- `POST /api/freeze-days` — добавить день или диапазон
- `DELETE /api/freeze-days/<date>` — удалить день фриза

### Статистика

- `GET /api/statistics/<team_id>` — статистика команды (0 = все команды)
- `GET /api/daily-stats/<team_id>` — ежедневная динамика (с параметрами `?start_date=&end_date=`)

## Конвенции, обнаруженные в коде

- **Форматирование кода:** комментарии-маркеры `@formatter:off` / `@formatter:on` (IDEA).
- **Работа с БД:** декоратор `@with_db_connection` для автоматического открытия/закрытия соединения, коммита и обработки ошибок.
- **Обработка ошибок БД:** параметры `default_return` и `raise_on_error` для гибкого управления.
- **Статусы задач:** константные строки `['new', 'planned', 'rollback', 'success']`.
- **Критичность:** `['high', 'medium', 'low']` с сортировкой в `get_tasks_by_team`.
- **ФИО:** поле `middle_name` опционально; форматирование через `utils.format_employee_name()`.
- **API:** все эндпоинты возвращают `jsonify`. Ошибки — с кодом 400/404 и полем `error`.
- **Шаблоны:** наследование через `{% extends "base.html" %}`, блоки `title`, `content`, `scripts`.
- **Стили:** один большой CSS-файл (1066 строк), BEM-подобная нотация классов.
- **Локализация:** интерфейс на русском языке.

## Разработка

- База данных создаётся автоматически при импорте `database.py`, если файл `database.db` отсутствует.
- Линтер/formatter не настроен; в проекте есть только `requirements.txt` с единственной зависимостью `Flask==2.3.3`.
- Проект использует `__pycache__/` и `*.db` в `.gitignore`.
- При удалении сотрудника, если у него есть назначения, они открепляются (employee_id → NULL), но не удаляются.
